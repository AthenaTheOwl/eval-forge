# AGENTS.md — eval-forge

Operating contract for AI agents working in this repo. Conventions
match the rest of the AthenaTheOwl portfolio.

## What this repo is

A CLI plus GitHub App that generates an eval pack from a target
repo's shape (RAG, agent, or hybrid), runs it on every PR, and
returns a pass / fail verdict to the CI gate. The framing is
deliberate: evals as gates, not dashboards.

The repo dogfoods itself. The eval suite for `eval-forge`'s own
code is generated by `eval-forge`, run in CI, and gates merges to
this very repo.

## Roles you may see in tasks

| Role | What they do |
|---|---|
| `pack-generator` | Reads a target repo, infers shape, emits an eval pack |
| `runner` | Executes a pack against a repo's current code; emits report |
| `gate` | Compares report against baseline; pass / fail decision |
| `template-curator` | Hand-curates per-shape template packs under templates/ |
| `app-bridge` | GitHub App glue that turns a webhook into a runner invocation |

These roles exist in spec ledger; not all are implemented in v0.

## Voice constraints

- The PR comment that the GitHub App posts is the customer surface.
  Read it through voice_lint before each commit.
- No marketing words. The eval delta speaks for itself.
- No antithetical-reversal phrasing.
- Customer-facing messages are flat. "Faithfulness dropped from
  0.92 to 0.81 on case `tier2-redacted`. PR blocked." Not florid.

## Gates (will land in spec 0002)

```bash
python -m pytest
python scripts/voice_lint.py
python scripts/validate_schemas.py
python -m eval_forge run evals/ --fail-on-regression
```

The last gate is dogfood: eval-forge gates eval-forge.

## Out of scope

- Hosted runner infrastructure. v0 ships the CLI and the GitHub App
  glue. Customers run the gate inside their own CI.
- Eval generation from LLM-driven analysis of arbitrary repos.
  Generation is template-driven off shape, plus customer
  configuration. The template plus config is what holds.
- Eval execution for non-Python target repos in v0. Adapters arrive
  later.
- Replacing dashboards. EvalForge is the gate; if a customer wants a
  dashboard they keep the one they have.
